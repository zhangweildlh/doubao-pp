## 四、发现的问题与修复

**问题根因**：360Chrome 的 MAIN world 内容脚本中 `window.chrome.runtime === undefined`，`bridgeEmit` 原用的 `chrome.runtime.sendMessage` 静默失败，background service worker 永远收不到桥接事件 → 记忆无法持久化。

**修复（commit `a90123f`，普通推送 `15b82ee..a90123f`）**：

1. 新增 `entrypoints/relay.content.ts`（ISOLATED world 中继）：监听 MAIN world 经 `window.postMessage` 投递的 `__doubaoPpRelay` 载荷，转发为 `__doubaoPpBridge` 桥接消息给 background。
2. `core/provider/doubao/dom-hook.ts`：`bridgeEmit` 改为双路径（优先 `chrome.runtime.sendMessage`，缺失时回退 `window.postMessage` 中继），同时兼容标准 Chrome 与 360Chrome。
3. `contracts.ts` / `dom-observer.ts` / `provider.ts` 同步修订校验与选择器；补充 `integration.dom-observer` 测试。

**修复后验证**：`mem_verify.py` 确认记忆持久化已恢复（见 3.3）。
